  - Uses Intersection Observer API for performance
  - 10% threshold for early triggering
  - Automatic cleanup on component unmount
  
- **Animation Variants**:
  - `data-animate` - Default fade-in with slide up
  - `data-animate="fade"` - Simple fade-in
  - `data-animate="slide-left"` - Slide from left
  - `data-animate="slide-right"` - Slide from right
  - `data-animate="scale"` - Scale up effect
  
- **Stagger Delays**:
  - `data-animate-delay="100"` to `500` (100ms increments)
  - Creates sequential reveal effect
  
- **Hover Interactions**:
  - Cards lift on hover (-2px transform)
  - Feature cards lift more (-4px) with enhanced shadow
  - Buttons have pulse animation effect
  - All with smooth 0.3s transitions

- **Accessibility**:
  - Respects `prefers-reduced-motion` setting
  - Disables animations for users who prefer less motion
  - Falls back to instant display

## Notes & Decisions

### Contact Form Implementation âœ…
**Chosen Solution:** EmailJS

**Why EmailJS:**
- No backend required - perfect for static/Jamstack sites
- 200 free emails/month
- Custom email templates
- Auto-reply capability
- Dashboard for tracking submissions
- Easy integration with client-side code

**Implementation Details:**
- Created composable pattern for reusability
- TypeScript interfaces for type safety
- Comprehensive validation rules
- Graceful error handling
- Professional UX with loading/success/error states

**Files Created/Modified:**
1. `/composables/useContactForm.ts` - Core form logic
2. `/components/ContactCta.vue` - Updated with form integration
3. `/nuxt.config.ts` - Added runtime config
4. `.env.example` - Environment variable template

**Required Setup (For Deployment):**
1. Create EmailJS account at https://dashboard.emailjs.com/
2. Create an email service (Gmail, Outlook, etc.)
3. Create an email template
4. Get Service ID, Template ID, and Public Key
5. Add to `.env` file (or hosting platform env vars):
   - NUXT_PUBLIC_EMAILJS_SERVICE_ID
   - NUXT_PUBLIC_EMAILJS_TEMPLATE_ID
   - NUXT_PUBLIC_EMAILJS_PUBLIC_KEY
